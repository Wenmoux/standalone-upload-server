## 本次优化内容

### 1. 转换结果 LRU 缓存

优化文件：`cirno-src/src/utils/chinese-convert.js`

新增小型 LRU 缓存：

- 缓存 key：`mode + 原始文本`
- 缓存范围：`simplified` / `traditional`
- 默认上限：800 条
- `none` 模式仍直接返回原文，不进入缓存

这个优化不改变任何繁简映射结果，只减少阅读页中同一段落反复渲染、滚动重绘、设置切换时的重复转换计算。

### 2. 保持转换准确性

本次没有盲目扩充词表。原因是当前 11 个大文本样本的标准 5 轮测试已经达到：

- 转换后异常残留：0
- 段落残留：0
- 分片残留：0
- 固定回归：全通过

因此优先优化稳定性和实际阅读性能，而不是增加可能误伤语义的规则。
